# 프로젝트 기본 지침

## 1. 기준 디바이스

본 프로젝트의 기본 작동 환경은 태블릿을 기준으로 한다.

화면 구성, 기획, 인터랙션, 여백, 버튼 크기, 정보 밀도, 터치 영역은 태블릿 사용성을 우선 고려하여 설계한다. 모바일 화면은 별도의 요구가 있을 때 대응하며, 기본 설계 기준으로 삼지 않는다.

## 2. 화면 및 기획 설계 방식

새로운 화면, 기능, 홈 구성, 학습 흐름을 설계할 때는 먼저 이미지 시안을 생성한다.

텍스트 설명만으로 바로 구현하지 않고, 이미지 시안을 통해 레이아웃, 정보 구조, 시각적 우선순위, 사용자 흐름을 먼저 확인한다. 이후 확정된 이미지 시안을 기준으로 실제 화면 작업을 진행한다.

이미지 시안은 단순한 참고 이미지가 아니라 구현 기준으로 사용한다.

## 3. 이미지 에셋 생성 방식

이미지 에셋이 필요한 경우, 기본적으로 각 에셋을 개별 이미지로 생성한다.

여러 요소를 하나의 큰 이미지로 묶어 처리하기보다, 학습 카드 이미지, 모드 아이콘, 보상 아이템, 캐릭터, 배경 요소, 장식 요소 등은 가능한 한 독립된 이미지 에셋으로 관리한다.

이를 통해 화면 구성 변경, 재사용, 교체, 반응형 대응이 쉽도록 한다.

## 4. 홈화면 설계 방향

홈화면은 초등학교 저학년 사용자가 직관적으로 이해할 수 있어야 한다.

사용자는 홈화면에 들어왔을 때 다음 두 가지를 바로 알 수 있어야 한다.

1. 오늘 어떤 학습을 할 수 있는지
2. 지금까지 얼마나 학습했는지

따라서 홈화면은 학습 진입 리스트와 학습 현황을 중심으로 구성한다.

- 학습 리스트의 1차 선택지는 콘텐츠팩을 기준으로 배치한다.
- 현재 기본 콘텐츠팩은 한자 학습과 수학 학습이다.
- 추후 영어, 과학, 국어, 역사 등 다른 콘텐츠가 추가될 수 있으므로 홈화면 구조는 확장 가능한 콘텐츠팩 목록을 기준으로 설계한다.
- 단어, 문장, 듣기, 쓰기, 복습, 퀴즈 같은 세부 학습 방식은 콘텐츠팩 내부 화면이나 콘텐츠팩 카드의 보조 정보로 다룬다.
- 각 콘텐츠팩은 짧은 이름, 쉬운 아이콘, 현재 상태, 이어하기 버튼을 함께 보여준다.
- 학습 현황은 오늘의 진행률, 연속 학습, 보상, 완료 항목, 복습 필요 항목처럼 아이가 바로 이해할 수 있는 정보로 구성한다.
- 한자 학습은 현재 핵심 콘텐츠 중 하나지만, 홈화면 전체가 한자 중심으로 보이지 않게 한다.
- 저학년 대상 서비스이므로 시각 요소는 충분히 친근해야 하지만, 장식이 학습 선택을 방해하지 않도록 한다.

## 5. 디자인 톤

디자인은 태블릿에서 보기 편한 어린이 학습 허브를 목표로 한다.

- 큰 터치 영역과 명확한 버튼을 사용한다.
- 텍스트는 짧고 쉽게 작성한다.
- 아이콘, 보상, 캐릭터, 색상 포인트를 활용해 친근함을 만든다.
- 화면은 밝고 부드러운 분위기를 유지한다.
- 장식 요소는 학습 리스트와 현황 정보를 보조하는 역할로 제한한다.
- 과도한 설명 문구, 복잡한 메뉴, 작은 글씨, 모바일 앱 하단 탭 중심 구조는 피한다.

## 6. 작업 순서

모든 화면 작업은 다음 순서를 기본으로 한다.

1. 태블릿 기준 사용 시나리오 확인
2. 이미지 시안 생성
3. 시안 기준으로 화면 구조 확정
4. 필요한 이미지 에셋 개별 생성
5. 구현 및 검수

## 7. 예외 처리

모바일 최적화, 데스크톱 대응, 통합 이미지 에셋 사용 등이 필요한 경우에는 별도 요청 또는 명확한 사유가 있을 때만 적용한다.

기본값은 항상 태블릿 기준 설계, 이미지 시안 우선, 개별 이미지 에셋 생성이다.

## 8. Git 반영 지침

사용자가 작업 결과를 "깃에 반영", "반영해줘", "깃 반영해줘"처럼 요청하는 경우, 기본적으로 로컬 커밋뿐 아니라 원격 저장소 푸시까지 진행한다.

별도 지시가 없는 한 다음 순서를 따른다.

1. 변경 파일 확인
2. 작업 범위에 해당하는 파일만 스테이징
3. 커밋 생성
4. 현재 브랜치의 원격 저장소로 푸시
5. 커밋 해시와 푸시 결과 공유

단, 작업 범위와 무관한 미추적 파일이나 사용자가 만든 것으로 보이는 변경 사항은 임의로 포함하지 않는다.

푸시를 원하지 않는 경우에는 사용자가 명시적으로 "커밋만 해줘", "푸시는 하지 마", "로컬에만 반영해"처럼 요청해야 한다.
