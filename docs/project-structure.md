# 프로젝트 구조 방향

이 저장소는 한자 전용 저장소가 아니라, 여러 학습 콘텐츠를 담을 수 있는 태블릿용 학습 게임 프로젝트로 관리한다.

## 권장 저장소 이름

아래처럼 특정 과목에 묶이지 않는 이름이 좋다.

- `learning-play-kit`
- `study-playground`
- `tablet-learning-games`
- `kids-study-games`
- `edu-card-games`

현재 앱의 화면 이름은 `한자 메모리`로 유지하되, 저장소 이름은 더 넓은 범위로 잡는 편이 좋다.

## 콘텐츠팩 개념

각 과목은 같은 데이터 구조로 관리할 수 있다.

```js
{
  id: "hanja-grade-8",
  title: "한자 8급",
  subject: "hanja",
  level: "8급",
  items: [
    {
      front: "山",
      back: "메 산",
      meta: { grade: "8급" }
    }
  ]
}
```

영어 단어라면 `front`는 영어 단어, `back`은 뜻이 된다. 수학이라면 `front`는 문제, `back`은 정답이 된다.

## 게임 모드 개념

게임 모드는 콘텐츠 종류와 최대한 분리한다.

- `memory`: 카드 짝 맞추기
- `choice-front-to-back`: 앞면을 보고 뜻/정답 고르기
- `choice-back-to-front`: 뜻/정답을 보고 앞면 고르기
- `ox`: 맞다/아니다 빠른 판별
- `review`: 틀린 항목 집중 복습
- `time-attack`: 제한 시간 점수 모드

이렇게 두면 한자뿐 아니라 영어, 과학 용어, 역사 인물에도 같은 게임 모드를 재사용할 수 있다.

## 단계별 전환 계획

1. 현재 단일 HTML 구조에서 한자 메모리 기능을 안정화한다.
2. 데이터와 게임 로직을 `콘텐츠팩`과 `게임 모드` 개념으로 분리한다.
3. 한자 외 샘플 콘텐츠팩을 하나 추가해 구조가 재사용 가능한지 확인한다.
4. 필요해지면 React/Vite 같은 웹앱 구조로 이전한다.

## 현재 유지할 원칙

- 태블릿에서 보기 쉬운 큰 터치 영역을 유지한다.
- 정적 파일만으로 실행 가능한 구조를 우선한다.
- 에셋 경로는 GitHub Pages와 로컬 실행을 모두 고려해 상대 경로를 사용한다.
- 특정 과목 UI 문구는 화면 단위에서만 쓰고, 저장소/공통 구조는 과목 중립적으로 관리한다.
